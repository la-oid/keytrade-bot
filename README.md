# KeyTrade Bot — руководство по эксплуатации

Документ для владельца проекта. Здесь описано, как развернуть систему на сервере Ubuntu, как менять настройки и тексты без помощи программиста.

---

## Что это такое

Проект состоит из **трёх ботов Telegram**, работающих с одной общей базой данных:

| Бот        | Для кого     | Что делает                                                |
|------------|--------------|-----------------------------------------------------------|
| **Buyer**  | Покупатели   | Принимает заказы на покупку ключей и оплату              |
| **Seller** | Продавцы     | Продавцы сдают ключи в паи и выводят заработок            |
| **Admin**  | Администраторы | Управление заказами, проверка оплат, выплаты, блокировки |

Все три бота запускаются одной командой и работают как единое приложение.

## 1. Развёртывание на сервере Ubuntu

### Что нужно

- Сервер с **Ubuntu 22.04** или новее (любой VPS подойдёт)
- IP-адрес сервера, логин и пароль (от хостинга)
- Минимум **1 ГБ ОЗУ** и **5 ГБ диска**
- Программа **PuTTY** на компьютере: [скачать](https://www.putty.org/)

### Шаг 1. Подключаемся к серверу через PuTTY

1. Запустите PuTTY
2. В поле **Host Name (or IP address)** введите IP сервера
3. Порт — `22`, Connection type — `SSH`
4. Нажмите **Open**
5. В чёрном окне введите логин (обычно `root`) → Enter
6. Введите пароль → Enter (символы пароля не отображаются — это нормально)

Все команды ниже выполняются в этом окне PuTTY.

### Шаг 2. Ставим необходимые пакеты

```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip git
```

### Шаг 3. Клонируем проект

```bash
cd /opt
sudo git clone https://github.com/la-oid/keytrade-bot.git
cd keytrade-bot
```

### Шаг 4. Создаём виртуальное окружение и ставим зависимости

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### Шаг 5. Создаём `.env`

```bash
cp .env.example .env
nano .env
```

Заполните файл по образцу из [раздела 2](#2-файл-env--главные-настройки). Сохранить в `nano`: `Ctrl+O`, `Enter`, выйти: `Ctrl+X`.

### Шаг 6. Проверяем, что всё запускается

```bash
python main.py
```

Если в Telegram все три бота отвечают на `/start` — всё работает. Останавливаем тестовый запуск: `Ctrl+C`.

### Шаг 7. Настраиваем автозапуск (systemd)

Чтобы бот работал постоянно и сам поднимался после перезагрузки сервера:

```bash
sudo nano /etc/systemd/system/keytrade-bot.service
```

Вставьте содержимое (замените `ИМЯ_ПОЛЬЗОВАТЕЛЯ` на ваше — узнать через `whoami`):

```ini
[Unit]
Description=KeyTrade Telegram Bots
After=network.target

[Service]
Type=simple
User=ИМЯ_ПОЛЬЗОВАТЕЛЯ
WorkingDirectory=/opt/keytrade-bot
ExecStart=/opt/keytrade-bot/venv/bin/python main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Сохраняем (`Ctrl+O`, `Enter`, `Ctrl+X`) и активируем:

```bash
sudo systemctl daemon-reload
sudo systemctl enable keytrade-bot
sudo systemctl start keytrade-bot
```

Проверяем статус:

```bash
sudo systemctl status keytrade-bot
```

Должна гореть зелёная надпись `active (running)`. Готово — бот работает.

## 2. Файл `.env` — главные настройки

Файл лежит в корне проекта (`/opt/keytrade-bot/.env`). Это **единственное место**, где хранятся токены и секретные данные.

### Пример содержимого

```env
# ── Telegram ───────────────────────────────────
TG_BUYER_TOKEN=871339452:AAGE9Dqk0ymAC37L7LkAUxECufKJAU
TG_SELLER_TOKEN=862540700:AAHSPtGr31bLKyio1yqodd70KUx7pVA
TG_ADMIN_TOKEN=878899800:AAHgkj5u7bNJdTSu7tTkI7698Syhzow
TG_ADMIN_IDS=[83163852, 512746523, 52598059]

# ── Приложение ─────────────────────────────────
APP_PAYMENT_URL=https://amethyst-malory-76.tiiny.site/
APP_SUPPORT_URL=https://t.me/Chain

# ── База данных ────────────────────────────────
DB_URL=sqlite+aiosqlite:///keytrade.db
```

### Что значит каждая строка

| Параметр           | Что это                                                                                                  |
|--------------------|----------------------------------------------------------------------------------------------------------|
| `TG_BUYER_TOKEN`   | Токен бота-покупателя. Берётся у [@BotFather](https://t.me/BotFather)                                    |
| `TG_SELLER_TOKEN`  | Токен бота-продавца                                                                                      |
| `TG_ADMIN_TOKEN`   | Токен админ-бота                                                                                         |
| `TG_ADMIN_IDS`     | Список Telegram ID админов в квадратных скобках через запятую. Узнать ID: [@userinfobot](https://t.me/userinfobot) |
| `APP_PAYMENT_URL`  | Ссылка на страницу с платёжными реквизитами. Меняется при смене платёжки                                 |
| `APP_SUPPORT_URL`  | Ссылка на аккаунт поддержки (куда жалуются пользователи)                                                |
| `DB_URL`           | Путь к базе данных. **Не меняйте**, если не понимаете, что делаете                                       |

### Как добавить нового админа

Откройте `.env`:

```bash
nano /opt/keytrade-bot/.env
```

Найдите строку `TG_ADMIN_IDS` и добавьте новый ID через запятую:

```env
TG_ADMIN_IDS=[831693852, 5120746523, 525988059, 999888777]
```

Сохраните и перезапустите бот:

```bash
sudo systemctl restart keytrade-bot
```

> ⚠️ **После любого изменения `.env` нужен перезапуск.** Иначе изменения не применятся.

## 3. Управление ботом

Все команды выполняются на сервере через SSH.

| Действие              | Команда                                  |
|-----------------------|-------------------------------------------|
| Запустить             | `sudo systemctl start keytrade-bot`      |
| Остановить            | `sudo systemctl stop keytrade-bot`       |
| Перезапустить         | `sudo systemctl restart keytrade-bot`    |
| Посмотреть статус     | `sudo systemctl status keytrade-bot`     |
| Посмотреть логи (live)| `sudo journalctl -u keytrade-bot -f`     |
| Логи за последний час | `sudo journalctl -u keytrade-bot --since "1 hour ago"` |

> Чтобы выйти из режима живых логов — нажмите `Ctrl+C`.

## 4. Изменение текстов в ботах

Все тексты сообщений и кнопок хранятся в Python-файлах. Их можно редактировать вручную — программирования не требуется, нужно только не трогать кавычки и отступы.

### Где что лежит

```
app/bots/
├── admin_bot/texts/
│   ├── messages_texts.py    ← тексты сообщений админ-бота
│   └── buttons_texts.py     ← названия кнопок админ-бота
├── buyer_bot/texts/
│   ├── messages_texts.py    ← тексты сообщений бота-покупателя
│   └── buttons_texts.py     ← названия кнопок бота-покупателя
└── seller_bot/texts/
    ├── messages_texts.py    ← тексты сообщений бота-продавца
    └── buttons_texts.py     ← названия кнопок бота-продавца
```

### Пример: меняем приветствие в боте-покупателе

Открываем файл:

```bash
nano /opt/keytrade-bot/app/bots/buyer_bot/texts/messages_texts.py
```

Находим:

```python
class MenuTexts:
    START_TEXT = "Добрый день! Вас приветствует бот по продаже ключей."
```

Меняем строку в кавычках на новую:

```python
class MenuTexts:
    START_TEXT = "Здравствуйте! Это лучший магазин ключей в сети."
```

Сохраняем (`Ctrl+O`, `Enter`, `Ctrl+X`) и перезапускаем:

```bash
sudo systemctl restart keytrade-bot
```

### Что в тексте можно использовать

- **Жирный**: `<b>текст</b>`
- **Курсив**: `<i>текст</i>`
- **Моноширинный** (для копирования): `<code>текст</code>`
- **Перенос строки**: `\n`
- **Переменные** в фигурных скобках — `{user_id}`, `{amount}` и т.п. — **не трогайте**, бот сам подставит значения

### ⚠️ Чего нельзя делать

- Удалять кавычки `"` или `'` вокруг текста
- Удалять или менять переменные в `{фигурных скобках}`
- Удалять отступы (пробелы в начале строки)
- Менять название переменной (то, что слева от `=`). Например, `START_TEXT = "..."` — `START_TEXT` не трогаем

Если случайно сломали файл, бот не запустится. В логах (`sudo journalctl -u keytrade-bot -f`) будет ошибка с номером строки — найдите её и исправьте.

## 5. Изменение констант

Константы — это числовые настройки, которые влияют на работу всей системы (цены, лимиты, тайминги). Все они в одном файле:

```bash
nano /opt/keytrade-bot/app/shared/constants.py
```

### Полный список с пояснениями

#### Цена и заказы

| Константа    | Текущее значение | Что значит                                                                  |
|--------------|------------------|-----------------------------------------------------------------------------|
| `KEY_PRICE`  | `50`             | Цена одного ключа в рублях. **Влияет на все расчёты** (покупки, выплаты)    |
| `MEDIUM`     | `{"min": 100, "max": 500, "step": 50}` | Средний опт: от 100 до 500 ключей, шаг 50 (100, 150, 200…) |
| `LARGE`      | `{"min": 1000, "step": 100}` | Крупный опт: от 1000 ключей, шаг 100 (1000, 1100, 1200…). Без верхнего лимита |

**Пример:** хотите средний опт от 200 до 1000 с шагом 100. Меняем:

```python
MEDIUM = {"min": 200, "max": 1000, "step": 100}
```

#### Банки

| Константа | Текущее значение            | Что значит                                                |
|-----------|-----------------------------|------------------------------------------------------------|
| `BANKS`   | `["Сбербанк", "Тинькофф"]` | Список банков, между которыми пользователь выбирает оплату |

Чтобы добавить «Альфа-банк»:

```python
BANKS = ["Сбербанк", "Тинькофф", "Альфа-банк"]
```

#### Крипто-наценка

| Константа                   | Текущее значение | Что значит                                                                  |
|-----------------------------|------------------|------------------------------------------------------------------------------|
| `CRYPTO_RATE_MARKUP_BUYER`  | `2`              | Прибавляем к курсу USDT для покупателей (курс ЦБ + 2 ₽). **Ваша прибыль с покупки** |
| `CRYPTO_RATE_MARKUP_SELLER` | `6`              | Вычитаем из курса USDT для продавцов (курс ЦБ − 6 ₽). **Ваша прибыль с выплаты**    |

#### Паи (тендеры)

| Константа                | Текущее значение | Что значит                                                  |
|--------------------------|------------------|--------------------------------------------------------------|
| `PIE_KEYS_MIN`           | `300`            | Минимум ключей в пае                                         |
| `PIE_KEYS_MID`           | `1000`           | Средний размер пая (для генерации фейков)                    |
| `PIE_KEYS_MAX`           | `1800`           | Максимум ключей в пае                                        |
| `PIE_KEYS_STEP`          | `50`             | Шаг количества ключей в фейковых паях                        |
| `PIE_MIN_LIFETIME_HOURS` | `1`              | Минимальное время жизни пая (часов)                          |
| `PIE_MAX_LIFETIME_HOURS` | `4`              | Максимальное время жизни пая                                 |
| `PIE_FAKE_LOW_COUNT`     | `5`              | Сколько мелких фейковых паёв держать в системе               |
| `PIE_FAKE_HIGH_COUNT`    | `3`              | Сколько крупных фейковых паёв держать в системе              |

#### Прочее

| Константа              | Текущее значение | Что значит                                                                  |
|------------------------|------------------|------------------------------------------------------------------------------|
| `KEY_CHECK_DURATION`   | `15`             | Сколько секунд показывается «Идёт проверка ключей» в боте-продавце           |
| `CASHOUT_STEP`         | `50`             | Минимальный шаг суммы вывода (рубли). Сумма вывода должна быть кратна этому числу |
| `FIRST_OFFER_DELAY_HOURS`    | `4`        | Через сколько часов после регистрации нового покупателя ему придёт спецпредложение |
| `FIRST_OFFER_KEYS`           | `10`       | Сколько ключей в первом спецпредложении                                      |
| `FIRST_OFFER_DURATION_HOURS` | `48`       | На сколько часов действует спецпредложение                                   |

### После изменений — перезапуск

```bash
sudo systemctl restart keytrade-bot
```

## 6. Настройка крипто-сетей

Файл: `data/crypto_networks.json`. Здесь — список криптовалют, через которые принимается оплата и делается выплата.

```bash
nano /opt/keytrade-bot/data/crypto_networks.json
```

### ⚠️ Самое важное

В файле есть строки вида `"address": "YOUR_BTC_ADDRESS"` — это **заглушки**. Перед запуском их **обязательно** нужно заменить на реальные адреса ваших кошельков, иначе пользователи будут отправлять деньги в никуда.

### Структура одной записи

```json
{
  "id": "btc",
  "name": "Bitcoin (BTC)",
  "address": "YOUR_BTC_ADDRESS",
  "hash_format": "SHA-256 → HEX",
  "hash_length": 64,
  "buyer_order": 3,
  "seller_order": 2
}
```

| Поле           | Что значит                                                                                  |
|----------------|----------------------------------------------------------------------------------------------|
| `id`           | Внутренний идентификатор. **Не меняйте у существующих сетей**                                |
| `name`         | Как сеть называется в кнопках для пользователя                                               |
| `address`      | **Ваш кошелёк** — сюда покупатели переводят оплату                                           |
| `hash_format`  | Подсказка пользователю, в каком формате присылать хэш транзакции                             |
| `hash_length`  | Сколько символов должно быть в хэше. Бот проверит и не пропустит неверный                    |
| `buyer_order`  | Позиция кнопки в боте-покупателе (1 = первая). **`0` или удалить поле — сеть скрыта**       |
| `seller_order` | Позиция кнопки в боте-продавце. Аналогично                                                   |

### Как заполнить кошельки

Заменяем заглушки на реальные адреса:

```json
{
  "id": "btc",
  "name": "Bitcoin (BTC)",
  "address": "bc1qxy2kgdygjrsqtzq2n0yrf2493p83kkfjhx0wlh",
  ...
}
```

### Как скрыть сеть, не удаляя её

Допустим, временно не хотим принимать BTC у покупателей, но хотим оставить выплаты в BTC продавцам. Меняем `buyer_order` на `0`:

```json
{
  "id": "btc",
  "buyer_order": 0,
  "seller_order": 2,
  ...
}
```

### Как поменять порядок кнопок

В файле сети сейчас в таком порядке у покупателя: ETH (1), BSC (2), BTC (3), SOL (4), TRX (5), TON (6). Хотите поставить BTC первым? Меняем `buyer_order` у BTC на `1`, у ETH на `2`, у BSC на `3` — и так далее.

### Как добавить новую сеть

Скопируйте любой блок (от `{` до `}` включая запятую) и вставьте новый, поменяв все поля. Главное — поле `id` должно быть уникальным.

### ⚠️ JSON чувствителен к синтаксису

- Каждая запись отделяется запятой, **кроме последней**
- Все строки — в двойных кавычках `"..."`
- Числа — без кавычек

После изменений — перезапуск:

```bash
sudo systemctl restart keytrade-bot
```

Если файл сломался, бот не запустится — в логах будет ошибка вида `JSONDecodeError`. Проверить файл онлайн: [jsonlint.com](https://jsonlint.com/).

## 8. База данных

Все данные системы — пользователи, заказы, балансы, выплаты, история — хранятся в одном файле:


## 9. Частые проблемы

### Бот не отвечает на сообщения

```bash
sudo systemctl status keytrade-bot
```

Если статус `failed` или `inactive` — смотрим логи:

```bash
sudo journalctl -u keytrade-bot -n 50
```

В последних 50 строках обычно понятно, что сломалось.

### `Unauthorized` или `Token is invalid`

Неверный токен в `.env`. Получите новый у [@BotFather](https://t.me/BotFather) и впишите.

### `JSONDecodeError` в логах

Сломан `data/crypto_networks.json`. Проверьте на [jsonlint.com](https://jsonlint.com/) — он подсветит ошибку.

### `SyntaxError` в логах при запуске

Сломан один из текстовых файлов (обычно из-за пропавшей кавычки). В логе будет путь к файлу и номер строки — открываем и исправляем.

### Бот работал и вдруг перестал

```bash
sudo systemctl restart keytrade-bot
```

В 90% случаев помогает. Если нет — смотрим логи.

### Закончилось место на диске

```bash
df -h
```

Если `/` забит — скорее всего, разрослись логи journalctl. Чистим старше 7 дней:

```bash
sudo journalctl --vacuum-time=7d
```
