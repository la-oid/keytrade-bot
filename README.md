# KeyTrade Bot — руководство по эксплуатации

Документ для владельца проекта. Здесь описано, как развернуть систему на сервере Ubuntu, как менять настройки и тексты без помощи программиста.

---

## Что это такое

Проект состоит из **трёх ботов Telegram**, работающих с одной общей базой данных:

| Бот        | Для кого       | Что делает                                                  |
|------------|----------------|--------------------------------------------------------------|
| **Buyer**  | Покупатели     | Принимает заказы на покупку ключей и оплату                 |
| **Seller** | Продавцы       | Продавцы сдают ключи в паи и выводят заработок              |
| **Admin**  | Администраторы | Управление заказами, проверка оплат, выплаты, блокировки    |

Все три бота запускаются одной командой и работают как единое приложение.

## 1. Развёртывание на сервере Ubuntu

### Что нужно

- Сервер с **Ubuntu 22.04** или новее (любой VPS подойдёт)
- IP-адрес сервера, логин и пароль (от хостинга)
- Минимум **1 ГБ ОЗУ** и **5 ГБ диска**
- Программа **PuTTY** на компьютере: [скачать](https://apps.microsoft.com/detail/xpfnzksklbp7rj)

### Шаг 1. Подключаемся к серверу через PuTTY

1. Запустите PuTTY
2. В поле **Host Name (or IP address)** введите IP сервера
3. Порт — `22`, Connection type — `SSH`
4. Нажмите **Open**
5. В чёрном окне введите логин (обычно `root`) → Enter
6. Введите пароль → Enter (символы пароля не отображаются — это нормально)

### Шаг 2. Ставим необходимые пакеты

```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip git
```

### Шаг 3. Клонируем проект

```bash
cd /opt
sudo git clone https://github.com/la-oid/keytrade-bot.git
cd keytrade-bot
```

### Шаг 4. Создаём виртуальное окружение и ставим зависимости

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### Шаг 5. Создаём `.env`

```bash
cp .env.example .env
nano .env
```

Заполните файл по образцу из [раздела 2](#2-файл-env--главные-настройки). Сохранить в `nano`: `Ctrl+O`, `Enter`, выйти: `Ctrl+X`.

### Шаг 6. Заполняем `cookie.json`

Перед запуском обязательно заполните файл с куками для оплаты СБП. Подробно — в [разделе 7](#7-настройка-оплаты-через-сбп).

### Шаг 7. Проверяем, что всё запускается

```bash
python main.py
```

Если в Telegram все три бота отвечают на `/start` — всё работает. Останавливаем: `Ctrl+C`.

### Шаг 8. Настраиваем автозапуск (systemd)

```bash
sudo nano /etc/systemd/system/keytrade-bot.service
```

Вставьте (замените `ИМЯ_ПОЛЬЗОВАТЕЛЯ` на ваше — узнать через `whoami`):

```ini
[Unit]
Description=KeyTrade Telegram Bots
After=network.target

[Service]
Type=simple
User=ИМЯ_ПОЛЬЗОВАТЕЛЯ
WorkingDirectory=/opt/keytrade-bot
ExecStart=/opt/keytrade-bot/venv/bin/python main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Сохраняем и активируем:

```bash
sudo systemctl daemon-reload
sudo systemctl enable keytrade-bot
sudo systemctl start keytrade-bot
sudo systemctl status keytrade-bot
```

Должна гореть зелёная надпись `active (running)`.

## 2. Файл `.env` — главные настройки

Файл лежит в корне проекта. Заполните его перед первым запуском — скопируйте `.env.example` и замените заглушки реальными значениями.

| Параметр               | Что это                                                                                                  |
|------------------------|----------------------------------------------------------------------------------------------------------|
| `TG_BUYER_TOKEN`       | Токен бота-покупателя — берётся у [@BotFather](https://t.me/BotFather)                                   |
| `TG_SELLER_TOKEN`      | Токен бота-продавца — там же                                                                             |
| `TG_ADMIN_TOKEN`       | Токен админ-бота — там же                                                                                |
| `TG_ADMIN_IDS`         | Список Telegram ID администраторов в квадратных скобках через запятую. Узнать свой ID: [@userinfobot](https://t.me/userinfobot) |
| `APP_SUPPORT_USERNAME` | Username аккаунта поддержки **без @**. Пользователи увидят кликабельную ссылку                          |
| `DB_URL`               | Путь к базе данных. Для SQLite оставьте как есть: `sqlite+aiosqlite:///keytrade.db`                     |

После любого изменения `.env` нужен перезапуск бота.

## 3. Управление ботом

| Действие               | Команда                                                    |
|------------------------|------------------------------------------------------------|
| Запустить              | `sudo systemctl start keytrade-bot`                        |
| Остановить             | `sudo systemctl stop keytrade-bot`                         |
| Перезапустить          | `sudo systemctl restart keytrade-bot`                      |

## 4. Изменение текстов в ботах

### Где что лежит

```
app/bots/
├── admin_bot/texts/
│   ├── messages_texts.py    ← тексты сообщений
│   └── buttons_texts.py     ← названия кнопок
├── buyer_bot/texts/
│   ├── messages_texts.py
│   └── buttons_texts.py
└── seller_bot/texts/
    ├── messages_texts.py
    └── buttons_texts.py
```

### Пример: меняем приветствие

```bash
nano /opt/keytrade-bot/app/bots/buyer_bot/texts/messages_texts.py
```

Находим и меняем строку в кавычках:

```python
START_TEXT = "Добрый день! Вас приветствует бот по продаже ключей."
```

Сохраняем (`Ctrl+O`, `Enter`, `Ctrl+X`) и перезапускаем:

```bash
sudo systemctl restart keytrade-bot
```

### Что можно использовать в текстах

- **Жирный**: `<b>текст</b>`
- **Курсив**: `<i>текст</i>`
- **Моноширинный** (для копирования): `<code>текст</code>`
- **Перенос строки**: `\n`
- **Переменные** в `{фигурных скобках}` — **не трогайте**, бот подставит значения сам

### ⚠️ Чего нельзя делать

- Удалять кавычки вокруг текста
- Удалять переменные в `{фигурных скобках}`
- Менять название переменной слева от `=`
- Нарушать отступы

## 5. Замена картинок в ботах

Все картинки хранятся в папке `data/images/` и разбиты по ботам:

```
data/images/
├── buyer/
│   ├── start.png            ← приветственный экран
│   ├── profile.png          ← личный кабинет
│   ├── medium_wholesale.png ← экран среднего опта
│   ├── large_wholesale.png  ← экран крупного опта
│   └── order_pending.png    ← заказ в обработке
└── seller/
    ├── start.png            ← приветственный экран
    ├── profile.png          ← личный кабинет
    ├── orders.png           ← список доступных заказов
    ├── keys_accepted.png    ← коды успешно приняты
    └── payment_created.png  ← заявка на вывод создана
```

### Как заменить картинку

Просто загрузите новый файл **с тем же именем** в ту же папку — старый будет перезаписан.

Через PuTTY (или любой SFTP-клиент, например [FileZilla](https://filezilla-project.org/)):

```
Путь на сервере: /opt/keytrade-bot/data/images/
```

Требования к файлам:
- Формат: **PNG** или **JPG**
- Название и расширение должны совпадать с оригиналом (например, `start.png`)

После замены картинок **перезапустите бот**:

```bash
sudo systemctl restart keytrade-bot
```

### ⚠️ Чего нельзя делать

- Переименовывать файлы — бот не найдёт картинку и упадёт с ошибкой
- Удалять файлы без замены

## 6. Изменение констант

```bash
nano /opt/keytrade-bot/app/shared/constants.py
```

### Полный список

#### Цены

| Константа          | Значение | Что значит                                |
|--------------------|----------|-------------------------------------------|
| `KEY_PRICE_BUYER`  | `289`    | Цена одного ключа для покупателя          |
| `KEY_PRICE_SELLER` | `50`     | Выплата продавцу за один ключ             |

#### Объёмы заказов (buyer_bot)

| Константа | Значение | Что значит |
|-----------|----------|------------|
| `MEDIUM`  | `{"min": 50, "max": 200, "step": 25}` | Средний опт: от 50 до 200 ключей, шаг 25 |
| `LARGE`   | `{"min": 300, "max": 1000, "step": 25}` | Крупный опт: от 300 до 1000 ключей, шаг 25 |

#### Крипто-наценка

| Константа                  | Значение | Что значит                                              |
|----------------------------|----------|---------------------------------------------------------|
| `CRYPTO_RATE_MARKUP_BUYER` | `2`      | Прибавляется к курсу ЦБ для покупателей (ваша прибыль) |
| `CRYPTO_RATE_MARKUP_SELLER`| `6`      | Прибавляется к курсу ЦБ для продавцов при выплате      |

#### Дедлайны платежей (минуты)

| Константа                | Значение | Что значит                                                  |
|--------------------------|----------|--------------------------------------------------------------|
| `PAYMENT_DEADLINE_PAY`   | `15`     | Сколько минут у покупателя чтобы оплатить после получения ссылки |
| `PAYMENT_DEADLINE_HASH`  | `10`     | Сколько минут чтобы прислать хэш крипто-транзакции          |
| `PAYMENT_DEADLINE_PDF`   | `10`     | Сколько минут чтобы прислать PDF-квитанцию                  |
| `PAYMENT_DEADLINE_REVIEW`| `240`    | Сколько минут у админа на проверку оплаты                   |

#### Паи (тендеры)

`PIE_RANGES` — список диапазонов фейковых паёв. Каждый диапазон задаёт минимум, максимум ключей и сколько таких паёв держать одновременно:

```python
PIE_RANGES = [
    {"min": 25,  "max": 200,  "count": 5},   # мелкие — держать 5 штук
    {"min": 200, "max": 500,  "count": 1},   # средние — держать 1
    {"min": 500, "max": 1000, "count": 2},   # крупные — держать 2
]
```

| Константа                | Значение | Что значит                                             |
|--------------------------|----------|--------------------------------------------------------|
| `PIE_KEYS_STEP`          | `25`     | Шаг количества ключей при генерации фейковых паёв      |
| `PIE_MIN_LIFETIME_HOURS` | `1`      | Минимальное время жизни пая (часов)                    |
| `PIE_MAX_LIFETIME_HOURS` | `4`      | Максимальное время жизни пая                           |

#### Тендер (шкала заполненности)

| Константа                       | Значение  | Что значит                                                                 |
|---------------------------------|-----------|----------------------------------------------------------------------------|
| `TENDER_DEFAULT_TOTAL`          | `100000`  | Размер тендера по умолчанию — запускается при старте и когда очередь пуста |
| `TENDER_MAX_PIE_KEYS`           | `500`     | Паи с количеством ключей **выше** этого значения не засчитываются в тендер |
| `TENDER_STATUS_INTERVAL_HOURS`  | `12`      | Как часто (в часах) admin-бот присылает статус текущего тендера            |

#### Прочее

| Константа                    | Значение | Что значит                                                    |
|------------------------------|----------|---------------------------------------------------------------|
| `KEY_CHECK_DURATION`         | `15`     | Секунд показывается «Идёт проверка» в боте-продавце           |
| `CASHOUT_STEP`               | `500`    | Минимальный шаг суммы вывода. Сумма должна быть кратна этому  |
| `CASHOUT_MIN`                | `3000`   | Минимальная сумма вывода                                      |
| `FIRST_OFFER_DELAY_HOURS`    | `4`      | Через сколько часов после регистрации придёт спецпредложение  |
| `FIRST_OFFER_KEYS`           | `10`     | Сколько ключей в первом спецпредложении                       |
| `FIRST_OFFER_DURATION_HOURS` | `48`     | На сколько часов действует спецпредложение                    |

После изменений:

```bash
sudo systemctl restart keytrade-bot
```

## 7. Настройка крипто-сетей

Файл: `data/crypto_networks.json`

```bash
nano /opt/keytrade-bot/data/crypto_networks.json
```

### ⚠️ Самое важное — заполните адреса кошельков

В файле есть строки вида `"address": "YOUR_BTC_ADDRESS"` — это заглушки. **Замените их на реальные адреса ваших кошельков** до запуска, иначе деньги уйдут в никуда.

### Структура одной записи

```json
{
  "id": "btc",
  "name": "Bitcoin (BTC)",
  "address": "bc1qxy2kgdygjrsqtzq2n0yrf2493p83kkfjhx0wlh",
  "hash_format": "SHA-256 → HEX",
  "hash_length": 64,
  "buyer_order": 3,
  "seller_order": 2
}
```

| Поле            | Что значит                                                                              |
|-----------------|------------------------------------------------------------------------------------------|
| `id`            | Внутренний идентификатор. **Не меняйте**                                                |
| `name`          | Как сеть называется в кнопках для пользователя                                          |
| `address`       | **Ваш кошелёк** — сюда пользователи переводят оплату                                    |
| `hash_format`   | Подсказка пользователю, в каком формате присылать хэш транзакции                        |
| `hash_length`   | Сколько символов должно быть в хэше. Можно указать точное число (`64`) или диапазон (`"87-88"`) |
| `buyer_order`   | Позиция кнопки в боте-покупателе (1 = первая). `0` — сеть скрыта                       |
| `seller_order`  | Позиция кнопки в боте-продавце. `0` — сеть скрыта                                      |

### Как скрыть сеть не удаляя её

Поставьте `0` в нужном поле:

```json
{
  "id": "btc",
  "buyer_order": 0,
  "seller_order": 2
}
```

BTC не будет показываться у покупателей, но останется у продавцов.

### ⚠️ JSON чувствителен к синтаксису

- Каждая запись отделяется запятой, **кроме последней**
- Все строки — в двойных кавычках `"..."`
- Числа — без кавычек

После изменений — перезапуск бота.

## 8. Настройка оплаты через СБП

Оплата через СБП работает через ваш аккаунт на **market.csgo.com**. Для этого нужно авторизоваться на сайте и скопировать куки в файл `data/cookie.json`.

### Структура файла

```json
{
  "user_agent": "Mozilla/5.0 ...",
  "cookies": [
    {"name": "PHPSESSID", "value": "ВАШ_PHPSESSID"},
    {"name": "d2mid",     "value": "ВАШ_D2MID"}
  ]
}
```

### Как получить куки

1. Откройте браузер (Chrome или Brave) и войдите в аккаунт на [market.csgo.com](https://market.csgo.com)
2. Нажмите `F12` → перейдите на вкладку **Application** (Приложение)
3. В левом меню найдите **Cookies** → `https://market.csgo.com`
4. Найдите и скопируйте значения:
   - `PHPSESSID` — вставьте в поле `"value"` первой записи
   - `d2mid` — вставьте в поле `"value"` второй записи

### Как получить User-Agent

1. Откройте в том же браузере: [whatismybrowser.com/detect/what-is-my-user-agent](https://www.whatismybrowser.com/detect/what-is-my-user-agent/)
2. Скопируйте строку User-Agent и вставьте в поле `"user_agent"`

### Как обновить файл на сервере

```bash
nano /opt/keytrade-bot/data/cookie.json
```

Вставьте актуальные значения, сохраните и перезапустите бот.

### Сроки действия кук

| Cookie      | Срок действия  |
|-------------|----------------|
| `PHPSESSID` | ~7 дней        |
| `d2mid`     | ~14 месяцев    |

> ⚠️ Когда `PHPSESSID` протухнет — бот перестанет создавать платёжные ссылки и будет сообщать об ошибке. Обновите куку и перезапустите бот.

### Признак протухших кук

В логах появится строка:

```
PaymentSessionDead: HTTP 401 — cookies протухли
```

или

```
PaymentSessionDead: Cloudflare блок
```

## 11. Частые проблемы

### Бот не отвечает

```bash
sudo systemctl status keytrade-bot
sudo journalctl -u keytrade-bot -n 50
```

### `Unauthorized` или `Token is invalid`

Неверный токен в `.env`. Получите новый у [@BotFather](https://t.me/BotFather).

### `PaymentSessionDead` в логах

Куки на market.csgo.com протухли. Обновите `data/cookie.json` — [раздел 7](#7-настройка-оплаты-через-сбп).

### `JSONDecodeError` в логах

Сломан `data/crypto_networks.json`. Проверьте на [jsonlint.com](https://jsonlint.com/).

### `SyntaxError` в логах

Сломан один из текстовых файлов. В логе будет путь и номер строки — откройте и исправьте.

### Бот работал и вдруг перестал

```bash
sudo systemctl restart keytrade-bot
```

### Закончилось место на диске

```bash
df -h
sudo journalctl --vacuum-time=7d
```
